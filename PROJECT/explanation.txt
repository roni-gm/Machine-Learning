Penjelasan Lengkap Program

1) Tujuan
Program ini melakukan klasifikasi jenis transmisi mobil (Manual vs Automatic) dari dataset `cardata.csv`.

2) Alur Program yang Dibenahi (ringkasan)
- Memuat dataset `cardata.csv`
- Membersihkan kolom yang tidak relevan (`Car_Name` dihapus)
- Mengencode target `Transmission` jika bertipe teks
- Memisahkan fitur (`X`) dan target (`y`)
- Membagi data ke `X_train`, `X_test`, `y_train`, `y_test` (stratified split)
- Membangun `ColumnTransformer` untuk preprocessing:
  - `StandardScaler` untuk fitur numerik
  - `OneHotEncoder(handle_unknown='ignore')` untuk fitur kategorikal
- Membuat `Pipeline` yang menggabungkan `preprocessor` + `KNeighborsClassifier`
- Menjalankan `GridSearchCV` terhadap hyperparameter KNN di dalam pipeline
- Mengevaluasi model terbaik pada test set dengan metrik: Accuracy, Precision (macro), Recall (macro), F1 (macro)
- Menyimpan pipeline terbaik ke `best_knn_pipeline.joblib`
- Menyediakan contoh pembuatan data baru yang sesuai dengan urutan fitur untuk prediksi

3) Mengapa Perubahan Ini Penting
- KNN sensitif terhadap skala fitur: tanpa scaling, fitur yang memiliki rentang besar mendominasi perhitungan jarak.
- Jika preprocessing tidak dilakukan dalam pipeline, hasil cross-validation dan prediksi akhir menjadi tidak konsisten (data dilatih dengan transformasi, namun prediksi menggunakan data mentah atau sebaliknya).
- Menggunakan `Pipeline` memastikan preprocessing diterapkan identik selama cross-validation, pelatihan final, dan inferensi.
- Menggunakan `ColumnTransformer` memisahkan numeric/categorical sehingga setiap jenis fitur mendapat transformasi yang sesuai.

4) Cara Menjalankan Script Perbaikan
- Pastikan `cardata.csv` ada di folder `PROJECT` yang sama.
- Dari command line di folder `PROJECT`, jalankan:

```bash
python fix_knn_pipeline.py
```

- Script akan:
  - Menjalankan GridSearchCV (akan memakan waktu tergantung ukuran data dan jumlah kombinasi hyperparameter)
  - Menyimpan `best_knn_pipeline.joblib` di folder yang sama
  - Menyimpan `example_new_data.csv` sebagai contoh format input untuk prediksi

5) Catatan untuk Pengguna / Siswa
- Saat membuat contoh `new_data` untuk prediksi, pastikan kolom dan urutan sesuai dengan `X.columns` yang dicetak oleh script. Nilai numerik dan kategorikal harus bertipe sama seperti saat pelatihan.
- Jika dataset memiliki banyak kategori jarang, pertimbangkan untuk menangani kategori langka sebelum OneHotEncoding (mis. gabungkan kategori minoritas menjadi 'Other').
- Untuk interpretasi model, KNN tidak menyediakan koefisien fitur; gunakan metode lain (mis. PCA, analisis fitur) untuk memahami kontribusi fitur.

6) File yang Ditambahkan
- `fix_knn_pipeline.py` : script perbaikan dan demonstrasi pipeline.
- `explanation.txt`      : file penjelasan ini.
- `best_knn_pipeline.joblib` : akan dibuat setelah menjalankan script.
- `example_new_data.csv` : contoh input untuk prediksi.

7) Rekomendasi Lanjutan
- Simpan versi akhir notebook (commit) setelah memastikan pipeline bekerja.
- Tambahkan validasi silang yang lebih lengkap (mis. stratified k-fold) bila dataset tidak seimbang.
- Pertimbangkan evaluasi ROC/AUC jika relevan.
